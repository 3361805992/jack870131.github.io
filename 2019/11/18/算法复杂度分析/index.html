算法复杂度分析

算法复杂度分析

最近开始通过极客时间王争老师的 <<数据结构与算法之美>> 专栏重新复习数据结构与算法。打算通过写博客的方式记录下自己的学习重点与心得。

复杂度分析主要用于衡量算法的执行效率，其中又分为时间复杂度分析以及空间复杂度分析。

大 O 复杂度表示法

导论

先从简单的引导开始，理解大 O 是怎么得出的。

void calculate(int n) {
    int total = 0;
    for (int i = 0; i < n; i++) {
        total += i;
    }
}

从以上这段程序中我们可以判断出，第二行执行了 1 单位时间，而第三和第四行分别执行了 $n$ 次(除去 for 循环的初始条件 i = 0 执行了 1 单位时间)，所以总共需要 $(2n + 2)$ 个单位时间。

接下来继续分析:

void calculate(int n) {
    int total = 0;
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            total += (i + j);
        }
    }
}

接着以上套路，这段程序第二行执行了1单位时间，第三行执行了 $n$ 次 (除去 for 循环初始条件 int i = 0 执行了 1 单位时间)，需要 $n$ 个单位时间。第四和第五行分别执行了 ${n^2}$ 次 (除去 for 循环初始条件 int j = 0 执行了 $n$ 个单位时间)，所以需要 $2n^2$ 个单位时间。总共计算下来，总执行时间为 $T(n) = (2n^2 + 2n + 2) * 单位时间$ 。

接下来，就是怎么使用大 O 表示法。大 O 的公式如下:
$$
T(n) = O (f(n))
$$
T(n) 表示代码执行时间，n 表示数据规模，f(n) 表示每行代码执行的次数总和，而 O 则表示代码的执行时间 T(n) 与 f(n) 成正比。

大 O 表示法可将以上两个例子的时间复杂度的写法简化为 $T(n) = O(n)$ & $T(n) = O(n^2)$ 。

重点

使用大 O 表示时间复杂度有以下几个诀窍:

  • 只需要关注循环执行次数最多的那段代码即可，比如以上列举的两个例子中的循环部份代码。

  • 加法法则: 如果最后计算的是变量相加，则总时间复杂度就等于运算量级最大的那段代码的时间复杂度。

    Example

    int calculate(int n) {
    int total1 = 0;
    for (int i = 0; i < n; i++) {
        total1 += i;
    }

    int total2 = 0;
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            total2 += j;
        }
    }
    return total1 + total2;
}

    这段代码的复杂度最终用大 O 表示法为: $O(n^2)$ 。

  • 乘法法则: 嵌套代码的复杂度等于嵌套内外代码的乘积。

    Example

    int calculate1(int n) {
    int total = 0;
    for (int i = 0; i < n; i++) {
        total += i + calculate2(i);
    }
    return total
}
int calculate2(int n) {
    int total = 0;
    for (int i = 0; i < n; i++) {
        total += i;
    }
    return total;
}

    这段代码的复杂度最终用大 O 表示法为 $O(n^2)$ 。

复杂度量级

常见的复杂度量级分别有以下几种:

  • 常量级: $O(1)$
  • 对数级: $O(logn)$
  • 线性级: $O(n)$
  • 线性对数级: $O(nlogn)$
  • 平方级: $O(n^2)$
  • 立方级: $O(n^3)$
  • x 次方级: $O(n ^ x)$
  • 指数级: $O(2^n)$ -> 非多项式量级
  • 阶乘级: $O(n!)$ -> 非多项式量级

非多项式量级的算法问题叫做 NP (Non-Deterministic Polynomial) 问题。

$O(logn)$ & $O(nlogn)$

其中最不好分析的为 $O(logn)$ & $O(nlogn)$ 。在这边举个例子帮助理解:

int x = 2;
while (x <= n) {
    x *= 2;
}

这段代码中的 i 每循环一次就乘 2，当大于 n 时循环结束。由此我们可以得出:
$$
n = 2^x
$$
只要知道 x 值是多少，我们就能知道代码执行的次数，所以 x 就是我们要求的值:
$$
x = log_{2}n
$$
用大 O 表示时间复杂度就是 $O(log_{2}n)$ 。

$O(m + n)$ & $O(m * n)$

这种形式则是因为代码的复杂度由两个数据规模决定，举例:

int calculate(int m, int n) {
    int total1 = 0;
    for (int i = 0; i < m; i++) {
        total1 += i;
    }

    int total2 = 0;
    for (int i = 0; i < n; i++) {
        total2 += i;
    }

    return total1 + total2;
}

以上代码由于 m 和 n 都无法预先得知量级是多少，因此时间复杂度用大 O 表示为: $O(m + n)$ 。

空间复杂度分析

空间复杂度分析则比较直接，举例:

void space(int n) {
    int[] arr = new arr[n];
    for (int i = 0; i < n; i++) {
        arr[n] = i;
    }
}

第二行初始化了一个大小为 n 的 int 类型数组，第四行申请了一个 for 循环的初始条件变量 i，仅占用常量级空间，与数据规模 n 无关。因此，用大 O 表示法表达空间复杂度就为 $O(n)$ 。
